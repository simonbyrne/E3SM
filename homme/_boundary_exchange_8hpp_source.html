29 /*
30  * BoundaryExchange: a class to handle the pack/exchange/unpack process
31  *
32  * This class (BE) takes care of exchanging the values of one or more fields in the
33  * GP on the boundary of the elements with the neighboring elements. In particular,
34  * it takes care of packing the values into some buffers, performing the exchange
35  * (usually, this includes some MPI calls, unless running in serial mode),
36  * and then unpacking the values, accumulating them into the receiving elements.
37  * This process can be done for an arbitrary number of 2d fields (that is,
38  * no vertical levels) and 3d fields (with vertical levels). If you have a
39  * vector field of dimension DIM, you need to register DIM separate scalar
40  * fields. Internally, for each input field the BE object stores a bunch of
41  * separate Views, that view the input field at each element and component
42  * (if vector field). When the exchange method is called, ALL the stored
43  * fields are packed/exchanged/unpacked. Therefore, if you have two sets
44  * of fields that need to be exchanged at different times, you need to
45  * register them into two separate BE objects.
46  *
47  * The registration happens in three steps:
48  *
49  * - a call to set_num_fields, which sets the number of 1d, 2d and 3d fields
50  * that will be exchanged. Once this method is called, it cannot be
51  * called again, unless the method clean_up is called first.
52  * - a number of calls to one of more of the register_field(...), methods,
53  * which set the fields into the BE class. You cannot register more fields
54  * than declared in the set_num_fields call. However you can, if you want,
55  * register less fields, although this scenario is not tested, and may
56  * be buggy, so you are probably better off calling set_num_fields with
57  * the actual number of fields you are going to register. Note that you
58  * are not allowed to call register_field(...) before set_num_fields.
59  * NOTE: you are NOT allowed to register 1d fields together with
60  * 2d/3d fields. The idea is that 1d fields are exchanged only as
61  * min/max quantities (so they are not accumulated). Please, use
62  * two different BE objects for accumulation and for min/max.
63  * - a call to registration_completed, which ends the registration phase,
64  * and sets up all the internal structure to prepare for calls to
65  * exchange(). This method MUST be called BEFORE any call to exchange.
66  *
67  * This class relies on the MpiBuffersManager (BM) class for the handling of the buffers.
68  * See MpiBuffersManager header for more info on that. As explained above,
69  * you may have different BE objects, which are however never used at the
70  * same time. Therefore, it makes sense to reuse the same BM for all of them.
71  * For this reason, the BM can serve multiple 'customers'. The BE and BM
72  * classes are linked by a provider-customer relationship. It is up to the BE
73  * to register itself as a customer in the stored BM, and to unregister
74  * itself before going out of scope, to make sure the BM does not keep
75  * a dangling pointer to a non-existent customer. This is why BE's destructor
76  * automatically removes the 'this' object from the stored BM's customers list
77  * (assuming there is a stored BM, otherwise nothing happens).
78  *
79  * In order to work correctly, BE needs a valid Connectivity and a valid
80  * BM (both stored as shared_ptr). They can be set at construction time
81  * or later, via a setter method. There are only a few rules:
82  *
83  * - once one is set, you cannot reset it
84  * - the Connectivity must be set BEFORE any call to set_num_fields
85  * - the BM must be set BEFORE any call to registration_completed
86  *
87  */
